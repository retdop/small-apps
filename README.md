# small-apps

A collection of small experiments and projects, deployed as a static site via GitHub Pages.

Live at: **https://retdop.github.io/small-apps**

## Structure

```
small-apps/
├── index.html              # Landing page (vanilla HTML)
├── script.js               # Renders the project grid from projects.js
├── styles.css              # Landing page styles (dark GitHub-style theme)
├── projects.js             # Project registry — the single source of truth
├── projects/               # Self-contained project files
│   ├── task-logger.html    # Vanilla HTML/CSS/JS app (no build step)
│   ├── task-logger.css
│   ├── task-logger.js
│   ├── mots-fleches.html   # Entry point for the compiled React app
│   ├── mots-fleches.js     # Built artifact (committed, generated by CI)
│   └── mots-fleches.css    # Built artifact (committed, generated by CI)
└── mots-fleches-source-app/  # React + TypeScript source for Mots Fléchés
```

## Design decisions

### No root-level build tooling
The landing page and most apps are intentionally dependency-free vanilla HTML/CSS/JS. This keeps the site fast, simple to deploy, and easy to inspect without any build step. There is no `package.json` or bundler at the root.

### projects.js as the project registry
`projects.js` is the single place to register a project on the landing page. `script.js` reads this array at runtime and renders the card grid. Adding a project does not require touching any other file.

### Hybrid project types
Two patterns are supported:
- **Vanilla apps** (e.g. Task Logger): one or a few self-contained files in `projects/`. No build step, deployed as-is.
- **Compiled apps** (e.g. Mots Fléchés): a source app in its own subdirectory, built by Vite during CI/CD. The compiled output (`mots-fleches.js` / `mots-fleches.css`) is written to `projects/` and served as a static file.

### Built artifacts are committed to the repo
Compiled JS/CSS for React apps live in `projects/` and are committed. This means the site is always deployable from any checkout without running a build locally, and diffs show what changed in the bundle.

### Dark GitHub-style theme
`styles.css` uses CSS custom properties (`--bg-primary`, `--accent`, etc.) matching GitHub's dark palette. New apps should ideally follow the same visual language.

### Two-stage CI/CD
- **CI workflow** (runs on pull requests): builds the React apps and validates all HTML files with `html-validate`.
- **Deploy workflow** (runs on push to `main`): rebuilds everything, uploads the whole repo as a Pages artifact, and deploys to GitHub Pages.

## Adding a new project

### Vanilla app

1. Create your files in `projects/` (e.g. `projects/my-app.html`).
2. Register it in `projects.js`:

```js
{
  title: "My App",
  description: "A short description.",
  link: "projects/my-app.html",
  date: "2026-03"
}
```

3. Commit and push to `main`. The deploy workflow runs automatically.

### React/compiled app

1. Scaffold a new app in a subdirectory (e.g. `my-app-source/`).
2. Configure Vite to output a single named JS file and CSS file into `projects/`.
3. Add build steps to both `.github/workflows/ci.yml` and `.github/workflows/deploy.yml` (mirror the existing `mots-fleches` steps).
4. Create an HTML entry point in `projects/my-app.html` referencing the compiled assets.
5. Register it in `projects.js` as above.

## Local development

### Landing page / vanilla apps
No build step needed. Open `index.html` directly in a browser, or serve the root with any static server:

```sh
npx serve .
```

### Mots Fléchés (React app)
```sh
cd mots-fleches-source-app
pnpm install
pnpm dev
```

The dev server runs independently. To preview it integrated in the full site, run a build:

```sh
pnpm build
cp dist/mots-fleches.js ../projects/mots-fleches.js
cp dist/mots-fleches.css ../projects/mots-fleches.css
```

## Setup (first time)

1. Go to **Settings > Pages** in the GitHub repo.
2. Set source to **GitHub Actions**.
3. Push to `main` to trigger the first deployment.
